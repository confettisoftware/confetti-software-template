LineWatch Backend API

This is the backend API for LineWatch+ push notifications.

Available Endpoints:

POST /api/test-notification

Send a test push notification

Body: {"deviceToken": "your_device_token"}

POST /api/send-notification

Send a custom push notification

Body: {"deviceToken": "token", "title": "Title", "message": "Message", "rideName": "Ride", "waitTime": 15}

GET /api/debug-env

Debug environment variables (development only)

GET /api/fetch-wait-times

Fetch current theme park ride wait times

Query params:

  • parkId (optional) - Park ID to fetch wait times for
  • rideIds (optional) - Comma-separated list of ride IDs to filter
  • useRealApi (optional) - Set to 'true' to use real Queue Times API data

Examples:

  • /api/fetch-wait-times - Mock data (default)
  • /api/fetch-wait-times?useRealApi=true&parkId=2 - Real data from Thorpe Park
  • /api/fetch-wait-times?parkId=magic-kingdom&rideIds=space-mountain,haunted-mansion - Filtered mock data

GET /api/fetch-wait-times/parks

Get list of all available parks from Queue Times API

Returns: Complete list of parks with IDs, names, locations, and timezones

Attribution: Data provided by Queue-Times.com

POST /api/monitor-wait-times

Monitor wait times and send notifications when thresholds are met

Body: {"deviceToken": "token", "preferences": [{"rideId": "space-mountain", "maxWaitTime": 30}]}

Attribution: Wait time data provided by Queue-Times.com
